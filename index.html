스터디 소개

문제를 많이 푸는 것에 그치지 않고, 왜 이 풀이를 선택했는지 설명할 수 있는 실력을 목표로 합니다.

문제 해결력

백준과 AtCoder를 통해 알고리즘 유형별 풀이 경험을 쌓고, 낯선 문제에 대응하는 힘을 기릅니다.

구현력

C++ 기반 구현 연습과 주석 습관을 통해 읽기 쉬운 코드와 실전 구현 감각을 함께 키웁니다.

설명 능력

대면 발표를 통해 시간 복잡도, 알고리즘 선택 이유, 핵심 포인트를 논리적으로 전달하는 연습을 합니다.

진행 방식

1. 주간 활동

  • 매주 백준 문제 4~5문제 풀이
  • 매주 AtCoder Beginner Contest(ABC) 참여
  • 상황에 따라 문제 수와 난이도는 유동적으로 조정

2. 대면 발표

  • 풀이한 문제 중 핵심 문제 발표
  • 시간/공간 복잡도 설명
  • 사용 알고리즘과 선택 이유 정리
  • 문제 해결 과정에서의 주안점 공유

3. ABC 발표 기준

  • 자신이 푼 문제 + 그 다음 문제까지 준비
  • A, B 문제는 간단 브리핑 수준으로 진행 가능
  • 풀이 유무와 관계없이 사고 과정 정리 권장

4. 코드 공유

  • 풀이 코드는 공용 GitHub 레포지토리에 업로드
  • 알고리즘 노트는 Notion에 정리
  • 세부 규칙은 첫 대면 이후 확정

운영 계획

초반 운영

백준에서 카테고리별 문제 풀이를 진행하며 기본 알고리즘과 자주 나오는 유형을 정리합니다.

5월 중순 ~ 말 이후

프로그래머스 문제 풀이로 범위를 넓혀 기업형 문제와 실전 응용 감각을 함께 익힙니다.

병행 학습

문제 풀이에 필요한 이론은 노션과 알고리즘 교재를 활용해 개인별로 정리하고 공유합니다.

추가 훈련

삼성 기출 문제를 주 1문제씩 풀어보며 낯선 구현형 문제에 익숙해지는 연습을 합니다.

스터디 규칙

기본 규칙

  • 모르는 문제라도 최소 30분 이상 고민해보기
  • 코드 작성 시 주석을 꼼꼼하게 달기
  • 단순 정답보다 풀이 과정과 근거를 중시하기

학습 태도

  • 틀려도 괜찮으니 적극적으로 시도하기
  • 본인이 이해한 내용을 말로 설명할 수 있도록 준비하기
  • GitHub와 Notion 정리를 통해 기록 남기기

첫 모임 안내

첫 모임에서는 간단한 OT, 그라운드룰 설정, 문제 선정 방식, 이론 정리 방식 등을 결정할 예정입니다.

OT 내용 요약

  • 매주 AtCoder ABC 참여
  • 5월 중순~말까지는 백준 카테고리별 문제 풀이 진행
  • 이후에는 프로그래머스 문제 풀이 예정
  • 이론 공부는 문제 풀이에 필요한 개념 위주로 노션 정리 + 교재 병행
  • 삼성 기출 문제를 1주일에 한 문제씩 풀이
  • 코드 작성 시 주석 처리 꼼꼼히 하기
  • 모르는 문제라도 최소 30분 이상 고민해보기
  • GitHub와 Notion이 만들어진 후 코드 및 알고리즘 노트 정리
